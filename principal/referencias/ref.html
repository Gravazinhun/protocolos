0. O que são protocolos de redes?

São um conjunto de regras e convenções que definem como a comunicação entre dispositivos em uma rede deve ocorrer. Eles garantem que os dados sejam transmitidos corretamente, de forma segura e eficiente entre os dispositivos conectados à rede, sejam eles computadores, servidores, roteadores, switches, ou outros dispositivos de rede.

3. Modelo OSI e TCP/IP

O modelo OSI (Open Systems Interconnection) e o modelo TCP/IP são dois modelos de referência utilizados para entender como as redes de computadores funcionam. Ambos definem como os dados devem ser transmitidos entre sistemas diferentes, mas têm algumas diferenças fundamentais em termos de estrutura e aplicação.

O modelo OSI é um modelo teórico composto por sete camadas que detalham como a comunicação deve ocorrer entre dispositivos em uma rede. Cada camada tem responsabilidades específicas e interage com a camada acima e abaixo dela para garantir que os dados sejam transmitidos de forma adequada. As sete camadas do modelo OSI são:
  • - Camada de Aplicação: Inclui protocolos que os usuários utilizam diretamente, como HTTP, FTP, SMTP, DNS, entre outros. Ela corresponde às camadas de aplicação, apresentação e sessão do modelo OSI.
  • - Camada de Transporte: A camada de transporte do TCP/IP corresponde à camada de transporte do modelo OSI e gerencia a comunicação de dados entre dispositivos, garantindo que os pacotes de dados cheguem corretamente e sem erros. O TCP e o UDP são os protocolos dessa camada.
  • - Camada de Internet: Também chamada de camada de rede, ela é responsável por endereçar e roteamento dos pacotes de dados. O protocolo principal dessa camada é o IP (Internet Protocol), mas também pode envolver o ARP (Address Resolution Protocol).
  • - Camada de Acesso à Rede: Essa camada lida com a transmissão física dos dados pela rede, lidando com os protocolos de enlace de dados e a camada física do modelo OSI, como Ethernet e Wi-Fi.

A principal diferença entre os modelos é que o modelo OSI é uma estrutura teórica e foi criado para padronizar as redes, enquanto o modelo TCP/IP é mais voltado para a prática e reflete o funcionamento real das redes, principalmente a Internet. O modelo OSI foi desenvolvido pela ISO (International Organization for Standardization) e tem como objetivo detalhar todos os aspectos da comunicação de rede, enquanto o modelo TCP/IP foi criado pela DARPA (Defense Advanced Research Projects Agency) e tem um foco mais direto na implementação de protocolos de rede na prática, especialmente a comunicação na Internet.

Em termos de aplicação, o modelo OSI oferece uma visão mais didática e detalhada dos processos de comunicação em rede, sendo útil para fins educacionais. Já o modelo TCP/IP é amplamente utilizado no dia a dia das redes, sendo o modelo real da Internet e utilizado por praticamente todos os sistemas conectados à web.
