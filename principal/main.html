HTTP(S)

HTTP - Hypertext Transfer Protocol

O que é HTTP?

O HTTP(HyperText Transfer Protocol) é o principal protocolo usado para enviar dados entre um navegador web e um site, tendo também sua versão mais segura o HTTPS(HyperText Transfer Protocol Secure), ele utiliza a porta 443 diferenciado-se de sua outra versão que utiliza a porta 80.

HTTPS - Hypertext Transfer Protocol Secure

O que é HTTPS?

É como o HTTP, porém, o HTTPS tem um sistema de maior segurança para ser utilizado, com a mensagem sendo transferida sem interceptações.

Como funciona?

Ele usa um protocolo de criptografia para guardar e proteger as comunicações(Transport Layer Security(TLS)), ele usa uma infraestrutura de chave publica assimétrica, normalmente esse sistema usa duas chaves diferentes para criptografar(chave pública: esta chave está disponível para todos que desejam interagir com o servidor de forma segura; chave privada: esta chave é controlada pelo proprietário de um site e é mantida, como o leitor pode imaginar, privada. Essa chave reside em um servidor web e é usada para descriptografar informações criptografadas pela chave pública). O HTTPS é fundamental na proteção de suas informações na web.

FTP - File Transfer Protocol

O que é o FTP?

>

FTP, ou File Transfer Protocol (Protocolo de Transferência de Arquivos), é um protocolo de rede utilizado para transferir arquivos entre um cliente e um servidor em uma rede de computadores, como a internet. Criado nos anos 70, ele é um dos métodos mais antigos e ainda amplamente usado para movimentar dados de um local para outro, especialmente em ambientes empresariais e de desenvolvimento de websites. Com o FTP, é possível enviar, baixar, renomear, excluir e organizar arquivos em um servidor remoto.

Como funciona?

O funcionamento do FTP se baseia em uma conexão entre dois pontos principais: um cliente (que normalmente é o dispositivo do usuário) e um servidor de arquivos. Para iniciar uma sessão FTP, o cliente se conecta ao servidor usando credenciais de login, como um nome de usuário e senha. Algumas sessões FTP permitem acesso anônimo, onde não são exigidas credenciais, mas isso depende das configurações de segurança do servidor. As transferências em FTP podem ser feitas no modo texto (ASCII) para arquivos de texto ou no modo binário para arquivos executáveis, imagens e outros formatos. O cliente FTP envia uma solicitação de transferência, e o servidor responde, concluindo a ação.

DNS - Domain Name System

O que é DNS?

DNS(Domain Name System ou Sistema de nome de domínio) ele traduz nomes de sites em endereços de IP, ele é como uma “agenda de contato”.

Como funciona?

Quando um usuário digita um domínio na barra de endereços do navegador, ocorre uma consulta DNS:

Consulta ao Cache Local, o sistema verifica se o endereço IP correspondente ao domínio já está armazenado no cache do dispositivo;

Servidor DNS Recursivo, Se não estiver no cache, a consulta vai para um servidor DNS recursivo, geralmente configurado pelo provedor de internet;

TCP - Transmission Control Protocol

Protocolo de comunicação orientado à conexão, garantindo a entrega dos pacotes de dados.

Operação: Divide os dados em pacotes, e garante que a transmissão seja realizada de forma ordenada e sem erros.

DHCP - Dynamic Host Configuration Protocol

Ele é responsável por distribuir os ips na rede de forma dinâmica e rápida.

POP3 - Post Office Protocol 3

É utlizado para receber e baixar e-mails de um servidor para um computador ou o dispositivo do usuário.

Ele não mantem os e-mails no servidor após o download, então, quando você baixa as mensagens, elas ficam armazenadas no seu computador, liberando espaço no servidor.

O POP3 não é ideal se você acessa e-mail de dipositivos diferentes, fazendo o IMAP melhor nesse quesito.

Um dos lados positivos, é que você pode acessar o e-mail de forma offiline, já que o e-mail foi armazenado de forma local.

SMTP - Simple Mail Transfer Protocol

O SMTP é como o "carteiro" que pega a sua mensagem de e-mail e a entrega ao destinatário.

Se o destinatário usa um provedor diferente, o SMTP transfere o e-mail do servidor de origem para o servidor de destino, onde fica aguardando até ser baixado ou visualisado.

É usado exclusivamente para enviar e-mails, para receber e gerenciar são necessários outros protocolos, IMAP e POP3

Atualmente, SMTP trabalha com métodos de autenticação e, na maioria das vezes, com criptografia para proteger as mensagens durante o envio.

IP(v4, v6)

IPv4 - Internet Protocol version 4

É como um endereço de internet composto por 4 números, sendo separados por pontos (exemplo: 192.168.1.1).

Cada número pode variar de 0 a 255.

Endereços curtos e limitados, foi amplamente usado, mas está se esgotando.

IPv6 - Internet Protocol version 6

É uma versão mais atual desses endereços, sendo composto por 8 grupos de 4 caracteres, separados por dois pontos.

Possui endereços suficientes para o mundo atual, e com algumas melhorias de segurança e eficiência.

TELNET

Protocolo de rede usado para acessar e gerenciar remotamente um computador ou um servidor, utilizando a internet ou rede local.

Ele permite que você entre em outro sistema e o controle como se realmente estivesse nele , utilizando comandos de texto.

TelNet não é criptografado, fazendo com que os dados transmitidos (incluindo senhas) possam ser interceptados, fazendo com que o SSH seja mais utilizado atualmente.

IMAP - Internet Message Access Protocol

É utilizado para acessar e gerenciar e-mails, mas diferente do POP3, ele baixa as mensagens para o seu dispositivo, permitindo que você gerencie diretamente do servidor.

Como os e-mails permanecem no servidor, qualquer alteração será refletida em todos os dispositivos que você acessa o e-mail.

Facilita a consulta e manutenção, pois permite que você organize seus e-mails em pastas diretamente no servidor.

Diferente do POP3, o IMAP só baixa o conteúdo do e-mail que você está utilizando, economizando espaço no dispositivo.

SSH - Secure Shell

Protocolo que permite acessar remotamente os computadores, geralmente para fins de admnistração e manutenção de servidores.

Mas diferente do TelNet, ele pode fazer esse acesso de uma forma segura, pois criptografa todos os dados transmitidos, protegendo senhas e comandos de uma só vez.

Depois de estar conectado, você pode utilizar o terminal para fazer comandos no sistema remoto, como se estivesse fisicamente presente.

Também permite a transferência de arquivos de uma forma segura entre seu dispositivo e o sistema remoto, usando o SCP(Secure Copy Protocol) ou SFTP(SSH File Transfer Protocol).
